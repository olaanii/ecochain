# Layout Components

This directory contains the core navigation and layout components for the EcoChain application.

## Components

### Breadcrumbs

Automatic breadcrumb navigation component that generates breadcrumb trails from the current route.

**Features:**
- ✅ Automatic route segment generation
- ✅ Predefined route label mapping
- ✅ Clickable breadcrumb segments for navigation
- ✅ Hides on top-level routes (/, /dashboard, /discover, /bridge)
- ✅ Responsive design (collapses on mobile)
- ✅ Accessibility compliant (ARIA labels, semantic HTML)

**Usage:**
```tsx
import { Breadcrumbs } from "@/components/layout/breadcrumbs";

export default function Page() {
  return (
    <div>
      <Breadcrumbs />
      {/* Your page content */}
    </div>
  );
}
```

**Props:**
- `maxItems?: number` - Maximum breadcrumb segments to display (default: 5)
- `separator?: React.ReactNode` - Custom separator element (default: ChevronRight icon)

**Examples:**

1. **Basic Usage:**
   ```tsx
   <Breadcrumbs />
   ```

2. **Custom Separator:**
   ```tsx
   <Breadcrumbs separator={<span>/</span>} />
   ```

3. **Limit Items:**
   ```tsx
   <Breadcrumbs maxItems={3} />
   ```

**Route Label Mapping:**

The component includes predefined labels for common routes:
- `/dashboard` → "Dashboard"
- `/discover` → "Discover"
- `/verification` → "Verification"
- `/verification/camera` → "Camera"
- `/verification/status` → "Status"
- `/merchants` → "Merchants"
- `/merchants/hub-main` → "Hub"
- `/merchants/redemption` → "Redemption"
- `/bridge` → "Bridge"

For unmapped routes, labels are auto-generated by capitalizing and formatting the URL segment.

**Demo Pages:**

Visit these pages to see breadcrumbs in action:
- `/merchants/hub-main` - Shows "Merchants → Hub"
- `/merchants/redemption` - Shows "Merchants → Redemption"
- `/verification/camera` - Shows "Verification → Camera"
- `/verification/status` - Shows "Verification → Status"

**Top-Level Routes (No Breadcrumbs):**

Breadcrumbs are hidden on these routes:
- `/` (landing page)
- `/dashboard`
- `/discover`
- `/bridge`

### TopNavBar

Primary navigation bar with authentication and wallet integration.

**Variants:**
- `landing` - Simplified public navigation for landing page
- `app` - Full navigation with authentication and wallet integration

**Usage:**
```tsx
import { TopNavBar } from "@/components/layout/top-nav-bar";

<TopNavBar variant="app" />
```

### MobileDrawer

Slide-out navigation drawer for mobile devices.

**Features:**
- Touch gesture support (swipe to close)
- Backdrop overlay
- Body scroll lock when open
- Accessibility (focus trap, keyboard navigation)

**Usage:**
```tsx
import { MobileDrawer } from "@/components/layout/mobile-drawer";

<MobileDrawer
  isOpen={isOpen}
  onClose={() => setIsOpen(false)}
  navItems={navItems}
/>
```

### SideNavBar

Secondary navigation sidebar for application pages.

**Usage:**
```tsx
import { SideNavBar } from "@/components/layout/side-nav-bar";

<SideNavBar navItems={navItems} collapsed={false} />
```

## Integration

All components are designed to work together seamlessly:

```tsx
import { TopNavBar, Breadcrumbs } from "@/components/layout";

export default function Layout({ children }) {
  return (
    <div>
      <TopNavBar variant="app" />
      <Breadcrumbs />
      <main>{children}</main>
    </div>
  );
}
```

## Styling

All components use the EcoChain design system with Tailwind CSS:
- Dark theme with semi-transparent backgrounds
- Accent color: `#cafd00` (lime green)
- Text colors: `#f3ffca` (light), `#adaaaa` (muted)
- Responsive breakpoints: `md:` (768px)

## Accessibility

All components follow accessibility best practices:
- Semantic HTML elements
- ARIA labels and attributes
- Keyboard navigation support
- Focus management
- Screen reader friendly
