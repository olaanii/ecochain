Design decisions / deviations from original 6-week plan
-------------------------------------------------------
1. Used Server-Sent Events instead of socket.io / ws.
   Reason: the app runs on Next.js 16 App Router without a custom server.
   SSE is natively supported by route handlers via ReadableStream, works
   with Edge/Node runtimes, and scales horizontally through Redis Pub/Sub.
   Client surface (`useServerEvents`) is identical from a product POV —
   if true bidirectional channels are needed later, a `/api/ws` with a
   custom server can be added without touching UI code.
2. Voting power: uses stakes' amount sum, falls back to rewards/100 to
   avoid zero-power accounts in testing. Replace with on-chain query
   once the EcoVerifier read path is wired.
3. `prisma as any` casts for `pushSubscription` — the repository uses a
   hand-rolled ambient `@prisma/client` shim (`src/types/prisma-client.d.ts`).
   Proper typing will come automatically once we drop the shim and rely
   on the generated client types. Low-priority cleanup.

How to verify
-------------
1. `pnpm prisma migrate deploy`
2. `pnpm dev` → visit /analytics, /leaderboard, /governance, /bridge.
3. Open two browser tabs on /governance → cast a vote in tab A → tab B
   should refresh instantly via SSE (no reload).
4. Click the bell in the top nav → enable notifications (requires VAPID
   env vars). A POST to /api/notifications/subscribe should succeed and
   a `PushSubscription` row appear in the DB.
